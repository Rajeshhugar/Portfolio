Agentic AI for News Classification & Alerts

End-to-end agentic pipeline that ingests news and social feeds, classifies risk in real time, and triggers prioritized WhatsApp alerts with full observability.

Project Overview

This system continuously ingests articles and posts from social media platforms and online news sources, stores them in a central SQL/NoSQL store, and passes new items to an AI Agent Processing Unit. A Classification & Risk Agent works with an LLM interface (GROQ/Llama) and a lightweight context/memory window to decide whether each item is urgent, relevant noise, or low-priority content. High-priority signals are formatted and delivered to users via WhatsApp, while everything is logged for auditability and future model improvements.

Problem Statement

Stakeholders need to react quickly to reputation-critical or risk-heavy news, but manual monitoring across multiple channels (X, Instagram, news portals, etc.) is slow, inconsistent, and noisy. Existing keyword-based alert systems often spam users with irrelevant updates and still miss subtle but important signals. The goal was to build an always-on, explainable agent that can filter, classify, and escalate news items with minimal human intervention.

Solution Implemented

The solution is built around an Agentic Processing Unit that sits between the data ingestion layer and downstream notification channels:

  • Data Ingestion Stage: Social and news data is aggregated by an external collector and stored as raw events in a SQL/NoSQL database.
  • Classification & Risk Agent: Subscribes to new rows, enriches them with context from a memory window (recent events, previous decisions), and queries an LLM (GROQ/Llama) for semantic classification and risk scoring.
  • Agent Decision Logic: Uses the model's output plus custom business rules to route each item into High Priority/Urgent or Low Priority/Spam.
  • Action & Response: High-priority items go through a message formatter and are sent via the WhatsApp API as concise, actionable alerts.
  • Observability & Logs: All decisions, deliveries, and errors are persisted into dedicated Agent Reasoning Logs, Delivery Logs, and Error Logs for monitoring, debugging, and offline evaluation.

This architecture turns a plain LLM call into a robust, production-ready agentic workflow with clear control points: ingestion, reasoning, decision, action, and observability.
